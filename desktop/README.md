# AEGIS — Desktop Edition (C++)

A self-contained native build of the AEGIS Field Security Operations Suite. One
executable embeds the entire web UI **and** a C++ backend, so there is no web
server, PHP, database server, or browser tab to manage — it runs as a local
application.

```
            ┌──────────────────────────────────────────────┐
            │  aegis-desktop  (single native executable)     │
            │                                                │
            │   native window (webview)  ─or─  system browser│
            │            │                                   │
            │            ▼                                   │
            │   http://127.0.0.1:<port>                      │
            │   ┌────────────────────────────────────────┐  │
            │   │ embedded HTTP server (cpp-httplib)       │  │
            │   │  • serves the AEGIS UI (embedded assets) │  │
            │   │  • /api/* JSON backend (same contract    │  │
            │   │    as the web edition)                   │  │
            │   │  • JSON-document store (atomic writes)   │  │
            │   └────────────────────────────────────────┘  │
            └──────────────────────────────────────────────┘
```

## What's different from the web edition

- **Single-user / local.** The OS user account is the trust boundary, so there
  is **no login** — the suite opens straight to the console. (The multi-user
  authentication in the web/server edition is unchanged and still available
  there.)
- **Storage** is a JSON document written atomically to the per-user data dir:
  - Windows — `%APPDATA%\AEGIS\aegis.json`
  - macOS — `~/Library/Application Support/AEGIS/aegis.json`
  - Linux — `$XDG_DATA_HOME/AEGIS/aegis.json` (or `~/.local/share/AEGIS`)
  - Override with the `AEGIS_DATA_DIR` environment variable.
- The same UI, theme switch, Spanish toggle, geocoding, link-analysis icons,
  and 13 data collections all work exactly as in the browser.

The backend implements the identical `/api/*` surface (`status`, `me`, `config`,
`collections/<name>` CRUD, `feeds`) so the front-end runs against it unmodified.

## Build

### Quick build (no dependencies, opens your browser)

Requires a C++17 compiler and Python 3 (for the asset-embed step).

```bash
cd desktop
./build.sh           # or:  make
./aegis-desktop      # starts the server and opens the suite in your browser
```

### Native-window build (CMake)

Opens in its own application window via [webview](https://github.com/webview/webview),
which CMake fetches automatically.

```bash
cd desktop
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DAEGIS_WITH_WEBVIEW=ON
cmake --build build -j
```

Platform runtime for the window:
- **Windows** — the [WebView2 runtime](https://developer.microsoft.com/microsoft-edge/webview2/)
  (preinstalled on Windows 11; the installer below can bootstrap it on Windows 10).
- **macOS** — WKWebView (built in).
- **Linux** — WebKitGTK + GTK3: `sudo apt install libwebkit2gtk-4.1-dev libgtk-3-dev`.

### Optional: live security feeds

The RSS aggregator needs an HTTPS client. Enable it with libcurl:

```bash
cmake -S . -B build -DAEGIS_WITH_CURL=ON      # add to the command above
```

Without it, the Security Feed page shows the curated source list with direct
links (it never errors). Sources are a fixed server-side allowlist — clients
never supply URLs, so there is no SSRF surface.

## Package for distribution

CMake's CPack produces a native installer per platform:

```bash
cmake --build build --target package
```

- **Windows** → `AEGIS-1.0.0-win64.exe` (NSIS installer) + a `.zip`
- **macOS** → `AEGIS-1.0.0-Darwin.dmg` (drag-to-Applications) + a `.tar.gz`
- **Linux** → `AEGIS-1.0.0-Linux.deb` + a `.tar.gz`

Code-sign and notarize per platform before wide distribution (macOS `codesign`
+ `notarytool`; Windows Authenticode). Build on each target OS — CPack does not
cross-compile installers.

## Run-time options

```
aegis-desktop [--port N] [--headless]
  --port N      bind a specific localhost port (default 8787, else an
                ephemeral free port if taken)
  --headless    run the server only; do not open a window or browser
                (useful for testing, or to point your own browser at the URL)
```

## Offline note

The maps (Leaflet/CARTO tiles), the link-analysis library (vis-network), and
the web fonts load from public CDNs, and geocoding uses OpenStreetMap Nominatim
— exactly as in the web edition. With no internet, those features degrade
gracefully (the app reports "library unavailable offline") while everything
local — logging, rosters, tasks, trips, config, backups — keeps working.

## Layout

```
desktop/
├─ src/main.cpp              the application (server + store + API + launcher)
├─ src/assets_generated.h    embedded web UI (regenerated by the build)
├─ third_party/httplib.h     cpp-httplib (MIT)        — vendored
├─ third_party/json.hpp      nlohmann/json (MIT)      — vendored
├─ tools/embed_assets.py     turns web/ into the embedded header
├─ web/                      the AEGIS UI copied from the web edition
├─ CMakeLists.txt            cross-platform build + CPack installers
├─ Makefile / build.sh       dependency-free build (browser launch)
└─ README.md
```
